# 4. 디지털 데이터의 저장과 표현

## 개요

- 본 문서는 디지털 데이터가 어떻게 저장되고 표현되는지에 대한 구조와 표현 방식을 정리한 학습 문서입니다.
- 디지털 포렌식에서는 데이터의 **저장 구조, 시간 정보, 위치 정보, 메타데이터**를 이해하는 것이 매우 중요합니다.

---

# 디지털 데이터와 수 체계

## 정의

- 디지털 데이터는 **0과 1의 두 가지 상태만을 사용하는 이진수(Binary) 기반 데이터**입니다.
- 데이터를 생성, 저장, 처리하는 모든 과정은 **바이너리 데이터 형태**로 이루어집니다.

> 디지털 데이터는 0과 1 두 상태만을 이용하여 정보를 표현하는 부호화 방식이다.

---

# 데이터 단위

- 디지털 데이터는 **물리적 단위**와 **논리적 단위**로 나누어 설명할 수 있습니다.

---

## 1. 물리적 단위

- 실제 저장장치에서 데이터를 저장하는 단위입니다.

| 단위 | 설명 |
|---|---|
| Bit | 0 또는 1 한 개 |
| Byte | 8개의 Bit |
| KB | 1024 Byte |
| MB | 1024 KB |
| GB | 1024 MB |
| TB | 1024 GB |

---

## 2. 논리적 단위

- 정보를 **구조적으로 저장하고 처리하는 단위**입니다.

| 단위 | 설명 |
|---|---|
| Field | 데이터 항목 |
| Record | 여러 Field의 묶음 |
| Block | 저장장치 입출력 단위 |
| File | 데이터 집합 |
| Database | 파일들의 집합 |

---

# 데이터 구조 (포렌식 분석에서 매우 중요)

## 데이터 구조 계층

```
Database
   └ File
        └ Record
             └ Field
```

---

# 데이터 구조 상세 설명

| 구조 | 설명 |
|---|---|
| Field | 여러 개의 바이트나 워드로 구성된 데이터 항목 |
| Record | 프로그램 내 자료 처리 기본 단위 |
| Block | 저장 매체에서 입출력되는 물리적 단위 |
| File | 다양한 데이터 묶음으로 구성된 응용 프로그램 처리 단위 |
| Database | 레코드 집합으로 조직화된 데이터 |

---

## Field

- 파일 구성의 최소 단위
- 하나의 데이터 항목

### 예

```
이름
나이
전화번호
```

---

## Record

- 여러 Field가 모여 하나의 레코드를 구성

### 예

```
이름 : 홍길동
나이 : 30
전화번호 : 010-1234-5678
```

---

## Block

- 저장매체 입출력 기본 단위
- 파일 시스템에서는 **섹터 또는 클러스터**

### 예

```
HDD Sector
SSD Page
File System Cluster
```

---

## File

- 여러 레코드와 블록이 모여 구성
- 응용 프로그램이 처리하는 기본 단위

> 디지털 포렌식에서 **파일은 가장 중요한 분석 대상이다.**

### 예

```
document.docx
image.jpg
log.txt
database.db
```

---

# 파일 구조 예시

```
File
 ├─ Record 0
 │   ├─ Field 1
 │   ├─ Field 2
 │   ├─ Field 3
 │   └─ Field N
 │
 ├─ Record 1
 │   ├─ Field 1
 │   ├─ Field 2
 │   ├─ Field 3
 │   └─ Field N
 │
 └─ Record N
     ├─ Field 1
     ├─ Field 2
     ├─ Field 3
     └─ Field N
```

# 시간 정보와 디지털 포렌식

## 시간 정보의 중요성

- 시간 정보는 **특정 행위가 발생한 시점**을 의미합니다.
- 디지털 포렌식에서는 사건의 **타임라인 재구성**에 매우 중요합니다.

### 예

- 파일 생성 시간
- 로그인 시간
- 웹 접속 기록
- 메시지 전송 시간

> 포렌식 분석의 핵심은 **시간 기반 사건 재구성(Timeline Analysis)**이다.

---

# 주요 시간 정보 표현 방식

---

# 1. ISO 8601

## 정의

- 국제 표준 날짜 표현 방식

## 형식

```
YYYY-MM-DDTHH:MM:SSZ
```

## 예시

```
2025-03-04T13:25:30Z
```

### 의미

```
2025년 3월 4일
13시 25분 30초
UTC 시간 기준
```

### 사용 예

- API
- 로그 시스템
- 데이터 교환 포맷

---

# 2. MS-DOS Time

## 정의

- FAT 파일 시스템에서 사용되는 시간 형식

## 특징

- 시간 정보가 **비트 단위로 저장됨**

### 구성

```
Date
Time
```

### 예시

```
2024-05-10
14:22:10
```

### 포렌식에서 사용 예시

- USB
- FAT32 파일 시스템

---

# 3. Unix Time

## 정의

- 1970년 1월 1일 00:00:00 UTC 이후 흐른 초

## 예

```
1710000000
```

### → 사람이 읽는 시간

```
2024-03-09 01:20:00 UTC
```

### 사용 예시

- Linux
- 서버 로그
- API 시스템

---

# 4. Windows FILETIME

## 정의

- 1601년 1월 1일부터 **100나노초 단위**로 저장되는 시간

## 특징

- 64bit 정수 사용

### 예시

```
133486560000000000
```

### 사용 예시

- NTFS 파일 시스템
- Windows 이벤트 로그

---

# 5. Webkit Time (Chrome Time)

## 정의

- Chrome 브라우저에서 사용하는 시간 형식

## 기준

```
1601년 1월 1일
```

## 단위

- 마이크로초

### 사용 예시

- Chrome 방문기록
- 브라우저 캐시
- 쿠키 기록

---

# Timestamp 분석 도구

## DCode Timestamp Decoder

- 디지털 포렌식에서 발견된 숫자형 timestamp를 **사람이 읽을 수 있는 시간으로 변환하는 도구**

### 다운로드 링크

```
https://www.digital-detective.net/dcode/
```

### 사용 예시

```
Unix Timestamp
Windows FILETIME
Chrome Timestamp
```

# 위치 정보와 디지털 포렌식

## 개념

- 위치 정보는 **행위가 발생한 장소 또는 이동 경로**를 나타냅니다.

### 예시

- 사진 촬영 위치
- 스마트폰 이동 경로
- 차량 이동 기록

```

### 사용 타원체

## 지역 좌표계

### 예시

Bessel 1841
```

---

# 지리 좌표계 vs 투영 좌표계

| 구분 | 설명 |
|---|---|
| 지리 좌표계 | 위도(latitude), 경도(longitude) |
| 투영 좌표계 | 평면 좌표 X,Y (미터 단위) |

# 정리

- 디지털 데이터는 이진수 기반으로 저장됩니다.
- 데이터는 물리적 단위와 논리적 단위로 구성됩니다.
- 포렌식 분석의 핵심 대상은 **파일(File)**입니다.
- 위치 정보는 행위의 공간적 위치를 파악하는 데 사용됩니다.
- 멀티미디어 파일의 EXIF 메타데이터는 중요한 디지털 증거가 됩니다.
